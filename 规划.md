# 一生的随机可对话记忆库 — 项目规划文档

> 像一个有记忆、会思考的私人收藏夹：所有碎片化的文字信息存入后，既能通过搜索一键取出，也能在加权随机的"抽卡"浏览中被意外串联，还能在 AI 的陪伴下不断生长出关联和理解。

---

#### 套件 1：录入套件 `InputKit`

**职责**：极速信息录入，最小打扰

| 组件 | 功能 |
|------|------|
| `QuickInput` | 首页输入框 + 粘贴/发送按钮 |
| `TagSelector` | 标签选择栏（展开/收起/多选） |
| `InfoAdder` | 信息附加面板（来源/所属组/补充信息） |
| `AISuggestPanel` | AI 辅助标签建议面板 |

**核心交互流程**：
```
用户输入 → 发送 → 入库成功 → 底部轻提示"已入库"
                        ↓
              输入框位置切换为 ["添加标签"] ["添加信息"]
                        ↓
              3秒后自动回退为输入框（用户无操作时）
```

**信息附加面板字段**：
- **来源**：支持自由输入 + 预设选择（在设置中预制）
- **所属组**：工作区概念，设置中预分类别
- **补充信息**：自由文本输入框

---

#### 套件 2：随机浏览套件 `RandomBrowser`

**职责**：加权随机"抽卡"浏览体验

**权重算法**：
```typescript
// 基础权重 = 1
// 星标权重 +1（星标为普通的2倍）
// 5天内使用(复制)过 +1（近期使用为普通的2倍）
// 倍数相加：星标+近期使用 = 4倍权重

function calculateWeight(entry: Entry): number {
  let weight = 1;
  if (entry.isStarred) weight += 1;
  if (isWithinLast5Days(entry.lastUsedAt)) weight += 1;
  return weight;
}
```

**范围筛选**：
- 按标签筛选（如只看 `#金句`）
- 按星标状态筛选（如只看未星标）
- 组合筛选（如 `#金句` 且未星标）

**交互设计**：
| 操作 | 响应 |
|------|------|
| 单击卡片 | 复制内容 + 更新 lastUsedAt |
| 长按卡片 | 呼出快捷菜单 |
| 左右滑动 | 切换上一张/下一张随机卡片 |

**快捷菜单选项**：
- 添加/编辑标签
- 切换星标
- 详情显示（加入时间等）
- 添加到组
- 查看连线

---

#### 套件 3：标签套件 `TagKit`

**职责**：标签的全生命周期管理

| 功能 | 说明 |
|------|------|
| 手动添加 | 自由输入，一条可挂多个标签 |
| AI 建议 | 选中条目 → AI 分析 → 推荐 3-5 个标签 → 用户确认添加 |
| 标签管理 | 合并、重命名、删除 |
| 标签选择栏 | 复用组件，支持搜索和多选 |

**AI 标签建议流程**：
```
用户选择"AI辅助" → 发送 {内容, 上下文, 提示词}
                        ↓
              AI 返回推荐标签列表
                        ↓
              选择栏更新选中状态（不显示信息）
                        ↓
              用户点击确定 → 保存标签
```

**提示词配置（Config）**：
```typescript
interface PromptConfig {
  tagSuggestion: string;      // 标签建议提示词
  relationSuggestion: string; // 关联建议提示词
  dialogueContext: string;    // 对话上下文提示词
  autoLink: string;           // 自动连线提示词
}
```

---

#### 套件 4：星标套件 `StarKit`

**职责**：星标状态管理与权重参与

- 二元状态：星标 / 非星标
- 独立于标签系统
- 在搜索、随机浏览、AI 对话中均可作为筛选维度
- 影响随机权重（2倍）

---

#### 套件 5：搜索套件 `SearchKit`

**职责**：多维度信息检索

**搜索维度**：
- 关键词（全文搜索）
- 标签筛选
- 星标状态筛选
- 组合条件

**结果交互**：
| 操作 | 响应 |
|------|------|
| 单击结果 | 一键复制 |
| 长按结果 | 呼出快捷菜单（同随机浏览） |

**扩展功能**：
- "存为智能标签"：将当前搜索条件保存为快捷入口

---

#### 套件 6：连线套件 `LinkKit`

**职责**：条目间关系网络构建

| 功能 | 说明 |
|------|------|
| 手动连线 | 长按"关联到…" → 搜索选择另一条目 → 建立双向关联 |
| AI 关系建议 | 选中两条条目 → AI 分析关系 → 生成描述 → 用户确认 |
| 连线展示 | 条目详情页显示"相关条目"列表 |

**连线与标签的区别**：
- 标签：有名字，可分类，描述"是什么"
- 连线：无名字，无分类，描述"与什么有关"

---

#### 套件 7：AI 对话套件 `AIChatKit`

**职责**：AI 全面渗透的对话能力

**四个渗透层面**：

1. **整理辅助**
   - 标签建议（见套件 3）
   - 自动关联建议：针对条目推荐可能相关的内容

2. **对话探索**
   - 选中条目 → "围绕此条对话" → 打开对话面板
   - 对话中可引用其他条目作为上下文

3. **主动推送**（可开关）
   - 录入新内容后："发现与 [某条目] 高度相关，需要连接吗？"
   - 浏览时：轻提示询问是否展开对话

4. **长期记忆**
   - `长时记忆.md` 配置文件
   - 包含：用户偏好、常用标签、关注领域、关联规则
   - AI 每次对话自动读取，随确认的关联不断更新

---

#### 套件 8：设置套件 `SettingsKit`

**职责**：全局配置与上下文管理

| 配置项 | 说明 |
|--------|------|
| AI API 配置 | Key、模型选择、自定义提示词 |
| DeepSeek 专属 | 针对 DeepSeek 模型的自定义参数 |
| 上下文范围 | 近期窗口 / 标签上下文 / 长效记忆 |
| 主动推送 | 开关 + 相似度敏感度阈值 |
| 数据管理 | 存储位置、备份/恢复、导出 |

**上下文范围设置**：
- **近期窗口**：最近 N 条录入作为隐形背景
- **标签上下文**：指定标签下的条目视为领域知识库
- **长效记忆**：启用/禁用长期记忆融入
- 以上三种可组合使用

---

#### 套件 9：数据层套件 `DataKit`

**职责**：数据持久化、同步与导出

**数据库表设计**：

```sql
-- 条目表
CREATE TABLE entries (
  id TEXT PRIMARY KEY,
  content TEXT NOT NULL,
  source TEXT,
  group_id TEXT,
  supplement TEXT,
  is_starred INTEGER DEFAULT 0,
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL,
  last_used_at INTEGER,
  copy_count INTEGER DEFAULT 0
);

-- 标签表
CREATE TABLE tags (
  id TEXT PRIMARY KEY,
  name TEXT UNIQUE NOT NULL,
  created_at INTEGER NOT NULL
);

-- 条目-标签关联表
CREATE TABLE entry_tags (
  entry_id TEXT,
  tag_id TEXT,
  PRIMARY KEY (entry_id, tag_id)
);

-- 连线表
CREATE TABLE links (
  id TEXT PRIMARY KEY,
  source_id TEXT NOT NULL,
  target_id TEXT NOT NULL,
  description TEXT,  -- AI 生成的关系描述
  created_at INTEGER NOT NULL
);

-- 组表
CREATE TABLE groups (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  sort_order INTEGER DEFAULT 0
);
```

**导出功能**：
- 全量导出（JSON / Markdown）
- 选择性导出（按标签/组/时间范围）
- 一键导出所有有用信息

**同步方案（后期）**：
- Cloudflare D1 作为云端数据库
- 冲突解决策略：最后写入优先 + 手动合并选项

## 3. MVP 分期规划（浏览检索优先）

### Phase 1：核心浏览检索（2-3 周）

| 套件 | 交付内容 |
|------|----------|
| 录入套件 | 极速输入 + 基础入库 + 底部提示 |
| 随机浏览 | 基础随机 + 卡片展示 + 复制功能 |
| 搜索套件 | 关键词搜索 + 结果列表 + 一键复制 |
| 数据层 | SQLite 建表 + CRUD + 基础导出 |

### Phase 2：标签星标体系（1-2 周）

| 套件 | 交付内容 |
|------|----------|
| 标签套件 | 手动添加 + 标签选择栏 + 标签管理 |
| 星标套件 | 星标切换 + 权重参与随机 |
| 随机浏览 | 权重算法 + 范围筛选 + 快捷菜单 |

### Phase 3：高级检索 + 信息录入（1-2 周）

| 套件 | 交付内容 |
|------|----------|
| 录入套件 | 信息附加面板（来源/组/补充） |
| 搜索套件 | 组合筛选 + 存为智能标签 |
| 设置套件 | 基础设置 + 数据导出 |

### Phase 4：AI 集成（2-3 周）

| 套件 | 交付内容 |
|------|----------|
| AI 对话 | 标签建议 + 对话面板 |
| 设置套件 | AI API 配置 + 提示词配置 |

### Phase 5：连线与关系网络（1-2 周）

| 套件 | 交付内容 |
|------|----------|
| 连线套件 | 手动连线 + AI 关系建议 + 连线展示 |

### Phase 6：智能增强（2 周）

| 套件 | 交付内容 |
|------|----------|
| AI 对话 | 主动推送 + 长期记忆 |
| 数据层 | Cloudflare D1 同步 |

## 5. 关键类型定义

```typescript
// types/index.ts

interface Entry {
  id: string;
  content: string;
  source?: string;
  groupId?: string;
  supplement?: string;
  isStarred: boolean;
  createdAt: number;
  updatedAt: number;
  lastUsedAt?: number;
  copyCount: number;
  tags?: Tag[];
}

interface Tag {
  id: string;
  name: string;
  createdAt: number;
}

interface Group {
  id: string;
  name: string;
  sortOrder: number;
}

interface Link {
  id: string;
  sourceId: string;
  targetId: string;
  description?: string;
  createdAt: number;
}

interface Settings {
  ai: {
    apiKey: string;
    model: string;
    baseURL: string;
    isDeepSeek: boolean;      // DeepSeek 专属选项
    deepSeekOptions?: {
      temperature?: number;
      maxTokens?: number;
    };
    prompts: PromptConfig;
  };
  context: {
    recentWindow: number;     // 近期窗口 N 条
    tagContext?: string;      // 标签上下文 ID
    enableLongTermMemory: boolean;
  };
  push: {
    enabled: boolean;
    similarityThreshold: number;
  };
}

interface PromptConfig {
  tagSuggestion: string;
  relationSuggestion: string;
  dialogueContext: string;
  autoLink: string;
}
```
