# yiyan-personal-database 新版规划 v0.2

> 更新时间：2026-07-22
> 基于对比分析结果重新制定，聚焦关键缺口修复 + 架构校正 + 功能补全

## 一、项目现状基线

**已完成基础：**
- 技术栈：React 18 + TypeScript + Vite 5 + Capacitor 6 + Zustand + 纯手写 UI
- 9 页面路由 + 双数据库实现（Web/Native）+ 3 Store + AI 服务层
- Phase 1-3 核心功能 ~90% 完成
- 数据管理器、导出、编辑页等超出原始规划的功能已实现

**核心缺口：**
- AI 功能层（标签建议/关系建议/对话）服务已写但 UI 全部未接入
- 连线手动创建 UI 缺失
- 设置页缺少多项配置 UI
- 搜索/标签管理部分功能缺失
- InfoAdder 组件孤立未使用

## 二、v0.2 开发路线图

### Sprint 1：AI 功能接入（优先级最高）

**目标：让已写好的 AI 服务层真正可用**

#### 1.1 RandomPage 接入 AIChatPanel
- 在 `RandomPage.tsx` 中引入 `AIChatPanel` 组件
- `QuickMenu` 的"AI 对话"按钮 `action` 改为 `onAIChat` 回调
- 新增 `AIChatPanel` 弹出层状态管理
- AIChatPanel 已实现完整对话逻辑，仅需接入

#### 1.2 TagSelector 接入 AI 标签建议
- `TagSelector` 的 `onAISuggest` 回调接入 `ai.suggestTags(entryContent)`
- 需要传入当前条目内容作为 AI 输入
- AI 返回标签名列表后，自动勾选匹配的已有标签，无匹配的高亮显示"新建"按钮
- 在 `HomePage` 和 `EntryEditPage` 的 TagSelector 实例中传入 `showAISuggestion={true}`

#### 1.3 连线 AI 关系建议
- 在 `LinkPage.tsx` 新增"AI 建议连线"按钮
- 调用 `ai.suggestRelation(currentEntry, recentEntries)`
- AI 返回条目 ID 列表 + 描述理由
- 展示为可点击列表，点击即创建连线

#### 1.4 首页录入后 AI 自动标签
- `HomePage.tsx` 新增条目后，可选调用 `ai.suggestTags(content)` 
- 将建议标签以可点击 chip 形式展示在"添加标签"选项上方
- 用户点击即自动添加，也可忽略手动选择

## 三、架构调整

### 3.1 UI 组件库决策

**现状：** 纯手写 SVG + CSS，未使用 Ant Design Mobile

**决策：** 维持现状，不引入 Ant Design Mobile

**理由：**
- 已有完整的 SVG 图标库和 glass morphism 风格
- 引入新 UI 库会导致风格冲突
- 当前 UI 质量足够，补全功能优先级更高
- 如果后续需要，可考虑提取为独立组件库

### 3.2 组件接入策略

| 孤立组件 | 接入位置 | 方式 |
|----------|----------|------|
| `InfoAdder` | `HomePage.tsx` | 替换当前"添加信息"跳转逻辑 |
| `AIChatPanel` | `RandomPage.tsx` | QuickMenu AI 对话按钮触发 |
| `ai.suggestTags()` | `TagSelector.tsx` | onAISuggest 回调 |
| `ai.suggestRelation()` | `LinkPage.tsx` | 新增 AI 建议按钮 |

### 3.3 类型定义补全

需要新增的类型：

```typescript
// 智能标签
interface SmartTag {
  id: string;
  name: string;
  queryJson: string;  // 序列化的搜索条件
  createdAt: number;
}

// 连线建议结果
interface RelationSuggestion {
  entryId: string;
  entryContent: string;
  description: string;  // AI 给出的关联理由
  confidence: number;
}
```
