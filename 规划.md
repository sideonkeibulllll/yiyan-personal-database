### Sprint 2.5：待办功能开发（v0.3 新增）

> **详细分析见 `分析.md`**

待办是与笔记平等的独立数据类型，独立存储、独立标签池、独立交互，但不进入随机池、不被普通搜索、不需要连线。

#### Phase A：类型定义 + 数据库建表 + Store（P0）

**新增类型（`src/types/index.ts`）：**
- `Todo`、`TodoTag`、`TodoTemplate`、`TodoTemplateItem`
- `TodoStatus`（pending/done）
- `TodoConfig`（倒计时配置 + 回收站配置）
- `TodoSearchTimeFilter`（future/expired/expiredOverMonth）
- `Settings` 新增 `todo` 字段

**新增数据库表（SQLite + Web localStorage）：**
- `todos` — 待办主表（含 `deleted_at` 回收站字段、`folder_date` 日期文件夹）
- `todo_tags` — 待办独立标签池（含 `color` 字段）
- `todo_tag_relations` — 待办-标签关联
- `todo_templates` — 模板
- `todo_template_items` — 模板中的待办项

**新增接口（`ITodoDatabaseService`）：**
- 待办 CRUD + 回收站（delete/restore/permanentDelete/emptyBin）
- 批量操作（batchUpdateTime/batchAddTags）
- 待办标签 CRUD
- 模板 CRUD + importTemplateToDate
- searchTodos(keyword, timeFilter)

**新增 Store：**
- `todoStore` — 待办状态管理
- `todoTagStore` — 待办标签状态管理

**预估工时：4-5h**

### 3.3 待办独立数据层（v0.3 新增）

待办采用完全独立的数据层：
- 独立 `todos` 表 + 独立 `todo_tags` 表 + 独立标签池
- 不复用 `entries` 表，不参与连线
- 不进入随机浏览池
- 普通搜索不搜待办（有待办专属搜索模式）
- 回收站用 `deleted_at` 字段实现（非独立表，性能足够）

## 四、数据库迁移

### v0.1 → v0.2

```sql
CREATE TABLE IF NOT EXISTS smart_tags (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  query_json TEXT NOT NULL,
  created_at INTEGER NOT NULL
);
```

### v0.2 → v0.3（待办功能）

```sql
-- 待办主表
CREATE TABLE IF NOT EXISTS todos (
  id TEXT PRIMARY KEY,
  title TEXT NOT NULL,
  note TEXT,
  status TEXT DEFAULT 'pending',
  start_time INTEGER,
  end_time INTEGER,
  is_today INTEGER DEFAULT 0,
  color TEXT,
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL,
  completed_at INTEGER,
  deleted_at INTEGER,
  folder_date TEXT
);

-- 索引
CREATE INDEX IF NOT EXISTS idx_todos_folder_date ON todos(folder_date);
CREATE INDEX IF NOT EXISTS idx_todos_deleted_at ON todos(deleted_at);

-- 待办标签
CREATE TABLE IF NOT EXISTS todo_tags (
  id TEXT PRIMARY KEY,
  name TEXT UNIQUE NOT NULL,
  color TEXT,
  created_at INTEGER NOT NULL
);

-- 待办-标签关联
CREATE TABLE IF NOT EXISTS todo_tag_relations (
  todo_id TEXT,
  tag_id TEXT,
  PRIMARY KEY (todo_id, tag_id),
  FOREIGN KEY (todo_id) REFERENCES todos(id) ON DELETE CASCADE,
  FOREIGN KEY (tag_id) REFERENCES todo_tags(id) ON DELETE CASCADE
);

-- 待办模板
CREATE TABLE IF NOT EXISTS todo_templates (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL
);

-- 模板待办项
CREATE TABLE IF NOT EXISTS todo_template_items (
  id TEXT PRIMARY KEY,
  template_id TEXT NOT NULL,
  title TEXT NOT NULL,
  note TEXT,
  start_time INTEGER,
  end_time INTEGER,
  is_today INTEGER DEFAULT 0,
  tag_ids TEXT,
  sort_order INTEGER DEFAULT 0,
  FOREIGN KEY (template_id) REFERENCES todo_templates(id) ON DELETE CASCADE
);
```

**迁移安全说明：**
- 全部 `CREATE TABLE IF NOT EXISTS`，不影响已有表
- 待办是全新功能，无旧数据迁移问题
- Web localStorage 使用独立 key（`yiyan_todos`、`yiyan_todo_tags` 等），与笔记数据隔离

## 八、技术风险与对策

### 8.1 滑动手势冲突（v0.3 新增）

**对策：** 水平 > 垂直时进入手势模式，阈值 80px。

### 8.2 倒计时性能（v0.3 新增）

**对策：** 独立组件 + 仅 < 60 分钟待办开启定时器。

### 8.3 时间轴着色性能（v0.3 新增）

**对策：** 一天待办量通常 ≤ 20，CSS 渐变/块拼接即可。

### 8.4 回收站查询性能（v0.3 新增）

**对策：** `deleted_at` + `folder_date` 建索引，待办数据量小。

### 8.5 待办数据与笔记数据隔离（v0.3 新增）

**对策：** 独立表 + 独立 localStorage key（`yiyan_todos` 等），普通搜索只查 `entries` 表。
