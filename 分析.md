# 待办功能（Todo）开发分析

> 生成时间：2026-07-23
> 最后更新：2026-07-23 13:15 GMT+8
> 基于：原始提示词2.md + 项目现状代码审查 + 用户多轮选择题确认
> 目的：构建与笔记平等的待办数据类型，独立存储但融入主库交互体系

---

## 一、定位与设计哲学

待办是与文字条目平等的另一种数据类型，但**独立存储、独立标签池、独立交互**：

- **独立数据层**：独立 `todos` 表 + 独立 `todo_tags` 表 + 独立标签池
- **不进入随机池**：待办不包含知识，不参与随机浏览
- **不参与普通搜索**：普通搜索只搜笔记，待办有专属搜索模式
- **不需要连线**：待办生命周期短，连线违反极简核心
- **独立标签池**：不与笔记共享标签
- **用「今日处理」代替星标**：待办的优先级用 `isToday` 标记
- **过期归档**：过期 1 个月的待办移到回收站（独立存储），不真正删除

## 二、数据模型设计

### 2.1 待办类型定义（`src/types/index.ts`）

```typescript
/** 待办状态 */
export type TodoStatus = 'pending' | 'done';

/** 待办实体 */
export interface Todo {
  id: string;
  title: string;              // 待办标题（纯文本）
  note?: string;              // 备注（自由文本，详细描述）
  status: TodoStatus;         // pending / done
  startTime?: number;         // 开始时间（可选，时间轴着色用）
  endTime?: number;           // 结束/目标时间（可选，倒计时用）
  isToday: boolean;           // 今日处理标记（代替星标）
  color?: string;             // 自定义颜色（可选，覆盖标签颜色）
  createdAt: number;
  updatedAt: number;
  completedAt?: number;       // 完成时间戳
  tags?: TodoTag[];           // 待办标签（独立标签池）
}

/** 待办标签 */
export interface TodoTag {
  id: string;
  name: string;
  color?: string;             // 标签颜色（时间轴着色用）
  createdAt: number;
}

/** 待办模板 */
export interface TodoTemplate {
  id: string;
  name: string;               // 模板名称（类似"组"）
  createdAt: number;
  updatedAt: number;
}

/** 模板中的待办项 */
export interface TodoTemplateItem {
  id: string;
  templateId: string;
  title: string;
  note?: string;
  startTime?: number;         // 相对时间偏移（毫秒）或绝对时间
  endTime?: number;
  isToday: boolean;
  tagIds: string[];           // 关联的待办标签 ID
  sortOrder: number;
}

/** 待办设置 */
export interface TodoConfig {
  /** 倒计时显示开关 */
  showCountdown: boolean;
  /** 倒计时显示格式 */
  countdownFormat: 'full' | 'compact' | 'daysOnly';
  /** 倒计时显示位置 */
  countdownPosition: 'aboveBottomNav' | 'pageTop' | 'floating';
  /** 删除确认弹窗开关 */
  confirmDelete: boolean;
  /** 回收站自动清理天数（0 = 不自动清理） */
  recycleBinRetentionDays: number;
}

/** 搜索时的待办时间筛选 */
export type TodoSearchTimeFilter = 'future' | 'expired' | 'expiredOverMonth';
```

### 2.3 数据库表结构

**SQLite（nativeDatabase.ts）新增表：**

```sql
-- 待办主表
CREATE TABLE IF NOT EXISTS todos (
  id TEXT PRIMARY KEY,
  title TEXT NOT NULL,
  note TEXT,
  status TEXT DEFAULT 'pending',
  start_time INTEGER,
  end_time INTEGER,
  is_today INTEGER DEFAULT 0,
  color TEXT,
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL,
  completed_at INTEGER,
  deleted_at INTEGER,           -- 回收站：非 NULL 表示已移入回收站
  folder_date TEXT              -- 所属日期文件夹，如 "2026-07-23"
);

-- 待办标签表（独立标签池）
CREATE TABLE IF NOT EXISTS todo_tags (
  id TEXT PRIMARY KEY,
  name TEXT UNIQUE NOT NULL,
  color TEXT,
  created_at INTEGER NOT NULL
);

-- 待办-标签关联表
CREATE TABLE IF NOT EXISTS todo_tag_relations (
  todo_id TEXT,
  tag_id TEXT,
  PRIMARY KEY (todo_id, tag_id),
  FOREIGN KEY (todo_id) REFERENCES todos(id) ON DELETE CASCADE,
  FOREIGN KEY (tag_id) REFERENCES todo_tags(id) ON DELETE CASCADE
);

-- 待办模板表
CREATE TABLE IF NOT EXISTS todo_templates (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL
);

-- 模板待办项表
CREATE TABLE IF NOT EXISTS todo_template_items (
  id TEXT PRIMARY KEY,
  template_id TEXT NOT NULL,
  title TEXT NOT NULL,
  note TEXT,
  start_time INTEGER,
  end_time INTEGER,
  is_today INTEGER DEFAULT 0,
  tag_ids TEXT,                -- JSON 数组
  sort_order INTEGER DEFAULT 0,
  FOREIGN KEY (template_id) REFERENCES todo_templates(id) ON DELETE CASCADE
);
```

**回收站策略：**
- 过期 1 个月的待办通过 `deleted_at` 字段标记移入回收站
- 回收站中的待办 `status` 保持不变，但不在常规列表中显示
- 回收站可在设置页查看和恢复/彻底删除
- `deleted_at` 为 NULL 表示正常待办，非 NULL 表示回收站中的待办
- 性能权衡：用 `deleted_at IS NULL` 过滤比单独表更简单，且数据量小（待办生命周期短）不会有性能问题

## 三、视图架构

### 3.1 页面路由

| 页面 | 路由 | 说明 |
|------|------|------|
| `TodoPage` | `/todo` | 待办日常页面（底栏第 3 项），日期切换 + 列表 |
| `TodoManagerPage` | `/todo/manager` | 待办管理器（时间轴视图 + 文件夹 + 批量操作） |
| `TodoEditPage` | `/todo/:id/edit` | 待办编辑页（添加信息用） |
| `TodoTemplatePage` | `/todo/templates` | 模板管理页 |
| `TodoTemplateEditorPage` | `/todo/templates/:id` | 单个模板的时间轴编辑页 |
| `TodoRecycleBinPage` | `/todo/recycle-bin` | 回收站页面 |

### 3.2 底栏调整

底栏从 4 项变为 5 项：`录入 / 随机 / 待办 / Chat / 设置`

### 3.3 待办页面（`TodoPage` `/todo`）

**日常查看/勾选的简洁列表页：**

- 顶部横向日期选择器：`[今天] [明天] [后天] [...]`
- 列表展示当天待办，竖排卡片
- 有目标时间且 < 60 分钟的待办右侧显示倒计时（精确到秒）
- 有目标时间但 ≥ 60 分钟的显示 `--:--`
- 未完成在上，已完成沉底（置灰+删除线）
- 左滑标记完成、右滑删除、下拉添加新待办
- 长按呼出菜单（编辑/目标时间/标签/删除）
- 底部倒计时条（受设置控制）

### 3.4 待办管理器（`TodoManagerPage` `/todo/manager`）

**时间轴视图 + 文件夹管理：**

#### 左侧时间轴
- 0 点 → 24 点，从上到下
- 已过去的时间统一灰色（比浅灰亮些）
- 未分配时间浅灰色

#### 右侧待办卡片
- 有标签的待办：卡片颜色 = 标签颜色
- 无标签的待办：卡片颜色 = 系统分配（按创建顺序循环色板，预设 5-8 种）

#### 连续块填充着色逻辑
- 一个待办如果有结束时间但没有开始时间，从上一个标志点（上一个待办结束或 0 点）到该待办结束时间，整段连续填充该待办的颜色
- 如果有开始时间和结束时间，则仅填充 [开始, 结束] 区间
- 两个待办之间的未分配时段显示为浅灰色

#### 文件夹
- 按日期组织：7月23、7月24、7月25...
- 过期 1 个月的自动移入回收站（`deleted_at` 标记），过期但未满 1 个月的沉底显示
- 点击文件夹进入该日期的待办列表

#### 批量操作
- 选中多个待办后底部出现操作栏：
  - 批量改时间：相对偏移（当前完成时间 + XX 秒）或绝对时间
  - 批量加标签：弹出待办专属标签选择器

### 3.5 模板系统

**入口：** 设置页 → 待办分区 → 模板管理

**流程：**
1. 新建模板 → 输入模板名称
2. 进入模板的时间轴编辑页面（与待办管理器时间轴一致）
3. 点击 + 号添加待办到模板
4. 保存模板

**使用模板：**
- 待办管理器某一天 → 「从模板导入」按钮 → 选择模板 → 批量导入
- 主页待办页面 → 「从模板导入」按钮 → 选择模板 → 批量导入

### 3.6 待办编辑页（`TodoEditPage`）

首页发送待办后 3 秒弹出的「添加信息」跳转到此页面。

**编辑字段：**
- 标题
- 开始时间 + 结束时间（快捷预设：今天/明天/下周/自定义 + 原生选择器）
- 今日处理标记（开关）
- 标签（待办独立标签选择器）
- 备注（自由文本）

## 六、倒计时展示

### 6.1 待办列表中的倒计时

待办页面 `/todo` 的列表项：
- 有目标时间且剩余 < 60 分钟 → 右侧显示精确倒计时（XX:XX:XX）
- 有目标时间但剩余 ≥ 60 分钟 → 显示 `--:--`
- 无目标时间 → 不显示

### 6.2 底部倒计时条

受设置控制（`todo.showCountdown`）：

- 格式：`距离 [待办标题] 还有 X天X时X分X秒`
- 多条倒计时时可左右滑动切换（一般 ≤ 3 条）
- 显示位置（设置可选）：
  - `aboveBottomNav` — 底栏上方（默认）
  - `pageTop` — 页面顶部
  - `floating` — 浮动悬浮
- 格式选项：
  - `full` — X天X时X分X秒
  - `compact` — X天X时
  - `daysOnly` — X天
- 无倒计时待办时不显示

## 八、完成状态处理

- 标记完成：`status='done'`，`completedAt=Date.now()`
- 视觉：置灰 + 删除线 + 沉底
- 已完成待办保留在当前视图底部，可重新激活
- 已完成待办可在搜索中找到

## 九、过期与回收站

### 9.1 过期处理

- 待办超过结束时间未完成 → 仍显示在列表中，但标记为"过期"（视觉提示）
- 过期但未满 1 个月 → 沉底显示
- 过期满 1 个月 → 移入回收站（`deleted_at = Date.now()`）

### 9.2 回收站

- 回收站中的待办不在常规列表中显示
- 回收站可在设置页 → 待办 → 回收站入口查看
- 回收站支持：恢复（`deleted_at = NULL`）/ 彻底删除
- 设置可配置自动清理天数（0 = 手动管理）
- 性能权衡：用 `deleted_at` 字段而非独立表，因为待办数据量小且查询时 `WHERE deleted_at IS NULL` 索引即可高效过滤

## 十、颜色分配规则

### 10.1 标签颜色

- 待办标签可指定颜色（`todo_tags.color` 字段）
- 有标签的待办：卡片颜色 = 标签颜色
- 多标签时：取第一个标签的颜色（或按标签优先级）

### 10.2 系统颜色

- 无标签的待办：按创建顺序循环分配系统色板
- 预设色板：5-8 种颜色，与 App 的 glass morphism 风格协调
  ```typescript
  const SYSTEM_COLOR_PALETTE = [
    '#806a4d',  // 铜色
    '#7a6650',  // 深铜
    '#9b8666',  // 浅铜
    '#6b5d4f',  // 暗棕

    '#5c5044',  // 深褐
    '#b8a387',  // 米棕
  ```

- 同一待办在生命周期内颜色不变（创建时分配，存入 `color` 字段）

## 十二、技术风险与对策

### 12.1 滑动手势冲突

**风险：** 待办列表左滑/右滑与页面滚动冲突

**对策：** 判断滑动方向，水平距离 > 垂直距离时才进入手势模式并禁用垂直滚动。滑动阈值 80px。

### 12.2 倒计时性能

**风险：** 每秒 `setInterval` 导致重渲染

**对策：** 倒计时条独立组件，`setInterval` 仅在组件内部，无倒计时不开启定时器。列表项倒计时同理，仅 < 60 分钟的待办开启。

### 12.3 时间轴着色性能

**风险：** 大量待办时时间轴渲染卡顿

**对策：** 一天的待办数量通常不会超过 20 条，时间轴用 CSS 渐变或 div 块拼接，无需虚拟滚动。

### 12.4 回收站查询性能

**风险：** `deleted_at IS NULL` 过滤在数据量大时变慢

**对策：** 为 `deleted_at` 和 `folder_date` 建索引。待办生命周期短，数据量小，不会有性能问题。
